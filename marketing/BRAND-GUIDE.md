## 4. Do's and Don'ts

### âœ… Do's

**Visual**
- Use high-quality fashion imagery
- Maintain consistent spacing (8px grid system)
- Apply subtle shadows for depth (0 4px 12px rgba(0,0,0,0.1))
- Include diverse representation
- Use gradient backgrounds sparingly for premium features
- Keep ample white space

**Messaging**
- Lead with benefits, not features
- Include the brand emoji signature ğŸ‘—ğŸ’âœ¨ in major communications
- Celebrate user wins publicly
- Use real user testimonials
- Acknowledge both crypto and fashion communities
- Be transparent about blockchain aspects

**Community**
- Respond to comments within 24 hours
- Feature user-generated content
- Acknowledge feedback publicly
- Share behind-the-scenes content
- Celebrate milestones together

### âŒ Don'ts

**Visual**
- Don't use stock photos that feel generic
- Don't clutter layouts with too many elements
- Don't use harsh, high-contrast shadows
- Don't stretch or distort the logo
- Don't use unlicensed imagery
- Don't deviate from the color palette

**Messaging**
- Don't oversimplify crypto (respect intelligence)
- Don't make fashion feel exclusive or snobby
- Don't promise unrealistic earnings
- Don't use all caps (except for single words in excitement)
- Don't ignore questions or concerns
- Don't use competitor names negatively

**Community**
- Don't ignore negative feedback
- Don't over-promote (80/20 value/promo ratio)
- Don't automate all responses
- Don't favor only high-earning creators
- Don't make blockchain feel scary or complex
